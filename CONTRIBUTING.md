# Contributing to Malarz Trzebnica

Dziękujemy za zainteresowanie współtworzeniem projektu!

## Jak Współtworzyć

### 1. Fork Repository

```bash
git clone https://github.com/your-username/malarz-trzebnica-php.git
cd malarz-trzebnica-php
```

### 2. Utwórz Branch

```bash
git checkout -b feature/my-new-feature
```

### 3. Wprowadź Zmiany

- Przestrzegaj PSR-12 coding standards
- Dodaj testy (jeśli dotyczy)
- Zaktualizuj dokumentację
- Commit z Conventional Commits format

```bash
git commit -m "feat(contact): Add spam protection"
```

### 4. Push i Pull Request

```bash
git push origin feature/my-new-feature
```

Następnie utwórz Pull Request na GitHub.

## Coding Standards

- PSR-4 autoloading
- PSR-12 coding style
- PHPDoc dla publicznych metod
- Komentarze w języku polskim

## Testing

```bash
composer test
composer lint
```

## Code Review

Wszystkie PR wymagają:
- [ ] Code review od co najmniej 1 osoby
- [ ] Przejście automated checks
- [ ] Brak konfliktów
